## ðŸ” 2. Generate a Signing Keystore

Android requires apps to be signed with a **release key**.

Run this command:

```bash
keytool -genkeypair -v -storetype PKCS12 -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
```

Move the keystore into:

```
android/app/my-release-key.keystore
```

---

## ðŸ”§ 3. Configure Gradle for Release Signing

### Edit `android/gradle.properties`:

```properties
MYAPP_UPLOAD_STORE_FILE=my-release-key.keystore
MYAPP_UPLOAD_KEY_ALIAS=my-key-alias
MYAPP_UPLOAD_STORE_PASSWORD=xxxxxx
MYAPP_UPLOAD_KEY_PASSWORD=xxxxxx
```

### Edit `android/app/build.gradle`:

```gradle
signingConfigs {
    release {
        storeFile file(MYAPP_UPLOAD_STORE_FILE)
        storePassword MYAPP_UPLOAD_STORE_PASSWORD
        keyAlias MYAPP_UPLOAD_KEY_ALIAS
        keyPassword MYAPP_UPLOAD_KEY_PASSWORD
    }
}
buildTypes {
    release {
        signingConfig signingConfigs.release
        minifyEnabled true
        shrinkResources true
        proguardFiles getDefaultProguardFile("proguard-android.txt"), "proguard-rules.pro"
    }
}
```

## ðŸ“Œ Common Interview Tips

### Q: APK vs AAB?

- **AAB is required by Google Play.**
- Play Store generates device-specific APKs (smaller install size).

### Q: Why minifyEnabled + shrinkResources?

- Reduce bundle size
- Remove unused code & resources

### Q: Why use release keystore?

- Ensures app authenticity
- Mandatory for publishing

### Q: What happens if I lose my keystore?

- You **cannot update the app**
- Must request Google Play to reset signing key (takes time)
