# React Native New Architecture: Fabric Architecture Explained (Interview Ready)

React Native's **New Architecture** introduces two major upgrades:

1. **Fabric** (New Rendering System)
2. **TurboModules** (New Native Module System)

This answer focuses on **Fabric Architecture**.

**Fabric** is the new rendering system in React Native that replaces the old rendering system (**Paper**).

Its purpose is to make UI updates:

- **Faster**
- **More predictable**
- **More memory-efficient**
- **More concurrent** (works smoothly with React 18 features)

Fabric solves limitations of the old architecture:

### 1. Asynchronous UI updates sometimes caused inconsistencies

### 2. No direct access to native UI

Old renderer wasn't designed for concurrent rendering.

**JSI** lets JavaScript call C++/native code directly without the old async bridge system.

**Benefit:** Faster JS â†” Native communication.

### âœ… 2. Synchronous UI Operations

Fabric allows JavaScript to synchronously access native UI elements when needed.

**Benefit:** More accurate, predictable UI updates.

### âœ… 3. Shadow Tree & Concurrent Rendering

Fabric uses a new shadow tree system that supports:

- Partial UI updates
- React 18 concurrent features (like `useTransition`)
- Better off-main-thread rendering

### âœ… 4. Yoga v3 Layout Engine

Improved performance and consistent layout behavior across platforms.

### âœ… 5. Better Memory Management

Fabric avoids JSON serialization and uses lightweight shared objects between JS and native.
