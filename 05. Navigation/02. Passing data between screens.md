## 1. Passing data using navigation.navigate()

The **most common method**.

```javascript
navigation.navigate('Profile', { userId: 12, name: 'John Doe' });
```

**Receive it:**

```javascript
function ProfileScreen({ route }) {
  const { userId, name } = route.params;
}
```

### When to use?

âœ” One-time or simple data transfer, no need to persist  
âœ” Navigate â†’ show details â†’ done

## 2. Using route.params with optional updates

If a screen needs to update the data and send back results:

```javascript

  user,
  onSave: (updatedUser) => setUser(updatedUser),
});
```


## 3. Using Global State (Context API, Redux, Zustand, Jotai)

Instead of passing params repeatedly, store data in global state.

## 5. Navigation state listeners

Useful when a screen refreshes based on incoming data.


## Comparison Summary

| Method                        | Best For                   | Example                       |
| ----------------------------- | -------------------------- | ----------------------------- |
| `navigation.navigate(params)` | Simple one-way data        | `navigate("Details", { id })` |
| Callback in params            | Send back modified data    | `onSave(updatedUser)`         |
| Context / Redux               | Shared global state        | Auth state, shopping cart     |
| Async Storage                 | Persist across restarts    | Token, preferences            |
| `useFocusEffect`              | Update when navigated back | Refresh list page             |
