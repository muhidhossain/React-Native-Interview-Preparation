## What is Deep Linking?

**Deep Linking** allows an app to be opened directly to a specific screen using a URL.
Just like clicking `https://facebook.com/profile` opens a specific page in a browser,
deep links open a specific route inside a mobile app.

### üìå Example:

```
myapp://product/42
```

‚Üí Opens the Product Details screen with product id = 42.

---

## Why Deep Linking is Useful?

| Use Case                            | Example                                       |
| ----------------------------------- | --------------------------------------------- |
| Opening specific screens from links | Email links ‚Üí Reset password screen           |
| App-to-App Navigation               | Opening Google Maps from Uber                 |
| From Push Notifications             | Clicking a notification ‚Üí Specific order page |
| QR Code / SMS Login Links           | OTP magic link authentication                 |

---

## Types of Deep Linking

| Type                                        | Works When App Closed? | Internet Required? | Example                        |
| ------------------------------------------- | ---------------------- | ------------------ | ------------------------------ |
| Basic Deep Linking                          | Yes                    | No                 | `myapp://home`                 |
| Universal Links (iOS) / App Links (Android) | Yes                    | No                 | `https://mydomain.com/profile` |

---

## Difference üî•

| Basic Deep Link                     | Universal/App Links                       |
| ----------------------------------- | ----------------------------------------- |
| Uses custom URI scheme (`myapp://`) | Uses website domain (`https://myapp.com`) |
| Easy to set up                      | More secure & preferred                   |
| Not SEO friendly                    | Works even if installed or not            |

## Configure Deep Linking in React Native

### 1Ô∏è‚É£ Define link configuration with React Navigation

```javascript
const linking = {
  prefixes: ['myapp://', 'https://myapp.com'],
  config: {
    screens: {
      Home: 'home',
      Product: 'product/:id',
    },
  },
};
```

### 2Ô∏è‚É£ Add Navigation Container

```javascript
<NavigationContainer linking={linking}>
  <Stack.Navigator>
    <Stack.Screen name="Home" component={HomeScreen} />
    <Stack.Screen name="Product" component={ProductScreen} />
  </Stack.Navigator>
</NavigationContainer>
```

### 3Ô∏è‚É£ Test using CLI

```bash
npx uri-scheme open myapp://product/10 --ios
npx uri-scheme open myapp://product/10 --android
```

## Universal / App Links Setup (Short Summary)

| Platform    | Files to configure                                                      |
| ----------- | ----------------------------------------------------------------------- |
| **iOS**     | `apple-app-site-association`, XCode ‚Üí Capabilities ‚Üí Associated Domains |
| **Android** | `assetlinks.json`, intent filters in `AndroidManifest.xml`              |

### Android example intent filter:

```xml
<intent-filter android:autoVerify="true">
    <action android:name="android.intent.action.VIEW" />
    <data android:scheme="https" android:host="myapp.com" />
    <category android:name="android.intent.category.BROWSABLE" />
    <category android:name="android.intent.category.DEFAULT" />
</intent-filter>
```
