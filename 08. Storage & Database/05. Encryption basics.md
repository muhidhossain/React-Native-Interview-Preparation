# Encryption Basics ‚Äî Securing Data in React Native

## ‚úÖ What is Encryption?

**Encryption** is the process of converting **plain text data** into **cipher text** so that unauthorized users cannot read it.
Only someone with the correct **key** can decrypt and access the original data.

üëâ **Goal:** Protect sensitive data (tokens, passwords, personal info)

---

## ‚úÖ Why Encryption Is Important in Mobile Apps

In React Native apps, sensitive data can be exposed through:

- Stolen devices
- Rooted / jailbroken phones
- Man-in-the-middle attacks
- Reverse engineering

**Encryption helps protect:**

- Authentication tokens (JWT, OAuth)
- User credentials
- Payment or personal data
- Cached offline data

---

## ‚úÖ Types of Encryption (Key Interview Topic)

### üîπ 1. Symmetric Encryption

- **Same key** used for encryption & decryption
- Fast and efficient
- Used for **local storage encryption**

**Example:**

- **AES (Advanced Encryption Standard)** ‚úÖ (most common)

```
Encrypt(data, secretKey) ‚Üí ciphertext
Decrypt(ciphertext, sameSecretKey) ‚Üí data
```

üìå **Used in React Native for:**

- Local database encryption
- Offline storage

---

### üîπ 2. Asymmetric Encryption

- Uses a **public key** and **private key**
- Public key encrypts, private key decrypts
- Slower but more secure for key exchange

**Example:**

- **RSA**
- **ECC**

üìå **Used for:**

- Secure key sharing
- SSL/TLS connections

## ‚úÖ Hashing vs Encryption (Interview Favorite)

| Encryption    | Hashing          |
| ------------- | ---------------- |
| Reversible    | Irreversible     |
| Uses keys     | No keys          |
| Protects data | Verifies data    |
| Example: AES  | Example: SHA-256 |

üìå **Passwords should be hashed, not encrypted**

---

## ‚úÖ Best Practices (Say This in Interviews ‚úÖ)

- ‚úî Never store secrets in plain text
- ‚úî Don't hardcode encryption keys
- ‚úî Use OS secure storage
- ‚úî Always use HTTPS
- ‚úî Rotate keys when possible
- ‚úî Server should handle heavy crypto, not client

---

## ‚úÖ Example Interview Answer (Short & Strong)
