## âœ… 1. Debouncing

### Definition

**Debouncing** ensures that a function runs only after a certain delay has passed without the user triggering the event again.

Think of it like:

> "Wait until the user stops doing somethingâ€¦ then run the function."

### Use cases in React Native

- **Search input**: call API only after user stops typing
- Form validation
- Prevent multiple button presses that fire repeatedly

### How it works

If the event triggers again within the delay, the timer resets.

### Example (React Native)

```javascript
import { useCallback } from 'react';
import debounce from 'lodash.debounce';

const handleSearch = useCallback(
  debounce((text: string) => {
    console.log('API Call with:', text);
  }, 500),
  []
);

<TextInput placeholder="Search" onChangeText={handleSearch} />;
```

âœ” The API is called only after 500ms of no typing  
âœ” Prevents unnecessary API calls

## âœ… 2. Throttling

### Definition

**Throttling** ensures a function runs at most once in a given time interval, regardless of how many times the event fires.

Think of it like:

> "Allow the function to run once every X milliseconds."

### Use cases in React Native

- **Scroll events** (`onScroll`)
- Continuous gesture events
- Tracking device orientation / accelerometer data
- Preventing multiple rapid submit button clicks

### How it works

If the function is called multiple times, only the first one in the interval executes, others are ignored until interval ends.

### Example (React Native)

```javascript
import throttle from 'lodash.throttle';

const handleScroll = throttle((event) => {
  console.log('Scroll position:', event.nativeEvent.contentOffset.y);
}, 200);

<ScrollView onScroll={handleScroll} scrollEventThrottle={16} />;
```

âœ” Prevents expensive actions from running too frequently  
âœ” Good for performance in scroll-heavy screens
