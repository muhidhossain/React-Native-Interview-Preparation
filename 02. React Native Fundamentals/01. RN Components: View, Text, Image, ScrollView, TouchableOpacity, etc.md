React Native provides a set of core, platform-independent components that render to native UI elements under the hood (e.g., `UIView` on iOS, `View` on Android). Understanding them is essential for interviews.

## ðŸŽ¯ 1. View

### What it is:

The most basic building block, similar to a `div` in web.

Used for layout, containers, wrappers, and structure.

### Key points:

- Supports **Flexbox** for layout.
- Can handle styles, touch events, nesting, and accessibility.
- Does NOT support text directly inside it on Android (must use `<Text>`).

### Common uses:

```jsx
<View style={{ padding: 20, backgroundColor: '#eee' }}>
  <Text>Hello</Text>
</View>
```

## ðŸŽ¯ 2. Text

### What it is:

Used to display text and inline formatting.

### Key points:

- Supports nested `<Text>` for styling parts of a sentence.
- Allows text touch events (e.g., `onPress`).
- Unlike web, you cannot put text directly inside `<View>` on Android.

### Example:

```jsx
<Text style={{ fontSize: 18 }}>
  Hello <Text style={{ color: 'blue' }}>React Native</Text>
</Text>
```

---

## ðŸŽ¯ 3. Image

### What it is:

Used to display images from local files or remote URLs.

### Key points:

- Requires explicit **width & height** for local images.
- Caches network images (but caching behavior differs per platform).
- `resizeMode` controls scaling (`contain`, `cover`, `stretch`, `center`).

### Example:

```jsx
<Image
  source={{ uri: 'https://example.com/photo.jpg' }}
  style={{ width: 120, height: 120, borderRadius: 60 }}
/>
```

## ðŸŽ¯ 4. ScrollView

### What it is:

A component that provides scrolling capability for long content.

### Key points:

- Renders **all items at once** â†’ bad for large lists.
- Best for **small static content** or forms.
- Supports **horizontal scrolling**.

### Example:

```jsx
<ScrollView>
  <Text>Item 1</Text>
  <Text>Item 2</Text>
  {/* Many more */}
</ScrollView>
```

**Interview Tip:** If they ask why use `FlatList` over `ScrollView` â†’ Because `FlatList` renders lazily, improving performance.

## ðŸŽ¯ 5. TouchableOpacity

### What it is:

A wrapper that makes content pressable with an opacity fade-out effect.

### Key points:

- Most commonly used pressable component.
- Provides visual feedback.
- Works across iOS and Android.

### Example:

```jsx
<TouchableOpacity onPress={() => console.log('Pressed')}>
  <Text>Press Me</Text>
</TouchableOpacity>
```

## ðŸŽ¯ 6. Additional Common RN Components You Should Know

### a) Pressable

A newer, more flexible alternative to touchables.

- **Recommended** over `TouchableOpacity` for modern apps.
- Allows styles to change based on press state.

```jsx
<Pressable
  onPress={() => {}}
  style={({ pressed }) => ({ opacity: pressed ? 0.5 : 1 })}
>
  <Text>Click</Text>
</Pressable>
```

### b) FlatList

High-performance list for large datasets.

- Supports lazy rendering, item recycling, and virtualization.

**Interview must-know:**

- `keyExtractor`
- `renderItem`
- `ListHeaderComponent` / `ListFooterComponent`

### c) SectionList

Like `FlatList` but with grouped sections.

### d) SafeAreaView

Prevents content from overlapping with device notches, status bar.

### e) TextInput

For user input fields.

- Has controlled/uncontrolled input patterns.

```jsx
<TextInput value={text} onChangeText={setText} placeholder="Enter your name" />
```

### f) Modal

Displays content on top of everything.

- Used for dialogs, popups.

### g) ActivityIndicator

Loading spinner.

### h) StatusBar

Manage status bar style (dark/light).
