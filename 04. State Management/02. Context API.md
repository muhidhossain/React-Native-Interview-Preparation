# Context API ‚Äî Global State Management

---

## üîπ What is the Context API?

The **Context API** is a built-in state management mechanism in React that allows data to be shared across the component tree without **prop-drilling** (passing props manually through every level).
It is useful when multiple components need access to the same data ‚Äî like authentication state, theme, user preferences, language, app settings, etc.

---

## üîπ Why do we need it?

Normally, React shares data top ‚Üí bottom using props.
If a deeply nested component needs that data, we pass it through every level ‚Äî even if some components don't use it.

**This problem is called prop-drilling.**

Context API solves it by allowing us to store data in a **global context**, so any component can access it directly.

---

## üîπ Key Components of Context API

| Part                         | Purpose                                              |
| ---------------------------- | ---------------------------------------------------- |
| `React.createContext()`      | Creates a context object                             |
| `Provider`                   | Supplies (provides) data to child components         |
| `Consumer / useContext hook` | Reads (consumes) the context value inside components |

---

## üîπ How it works (Step-by-step)

### 1Ô∏è‚É£ Create Context

```javascript
import { createContext } from 'react';

export const AuthContext = createContext();
```

### 2Ô∏è‚É£ Wrap your app with Provider

```javascript
import { AuthContext } from './AuthContext';

export default function App() {
  const [user, setUser] = useState(null);

  return (
    <AuthContext.Provider value={{ user, setUser }}>
      <RootNavigator />
    </AuthContext.Provider>
  );
}
```

Now any nested component can access `user` without prop drilling.

### 3Ô∏è‚É£ Consume using useContext

```javascript
import { useContext } from 'react';
import { AuthContext } from '../AuthContext';

export default function ProfileScreen() {
  const { user, setUser } = useContext(AuthContext);

  return <Text>Welcome {user?.name}</Text>;
}
```

## üîπ When to Use Context API?

Use when many components require the same shared data:

‚úî Authentication state  
‚úî Theme (dark/light mode) üåô  
‚úî User settings & preferences  
‚úî Language translation  
‚úî Cart data in e-commerce apps  
‚úî Global app config

---

## üîπ When NOT to Use Context API?

Context API may cause performance issues with large data updates.

‚ùå High-frequency changing data (e.g. real-time UI updates)  
‚ùå Very complex state logic

**For such cases, Redux/Zustand/Recoil might be better.**

---

## üîπ Context API vs Redux (Interview Point)

| Context API                         | Redux                          |
| ----------------------------------- | ------------------------------ |
| Built-in (no extra library)         | Requires external library      |
| Good for simple-medium global state | Good for large & scalable apps |
| No strict structure                 | Predictable + standardized     |
| Uses Provider + useContext          | Uses store, actions, reducers  |

**Expected interview question:**

> "When would you choose Context API over Redux?"  
> ‚Üí When the app needs simple global state without heavy logic.
