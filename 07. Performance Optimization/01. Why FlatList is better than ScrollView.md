# Why FlatList Is Better Than ScrollView

(React Native interview preparation)

---

## 1. Purpose

- **ScrollView**: Renders all child components at once, even if they are off-screen.
- **FlatList**: Designed for large lists and uses **lazy loading + virtualization**.

---

## Key Reasons FlatList is Better

### 1ï¸âƒ£ Performance (Virtualization)

**FlatList** renders only what is visible on the screen, plus a small buffer.

Items outside the viewport are not mounted, improving:

- Memory usage
- Rendering speed
- Overall app smoothness

**ScrollView** loads everything into memory, which becomes slow and heavy with long lists.

- **FlatList** can handle hundreds or thousands of items efficiently.
- **ScrollView** struggles with long lists and may cause:
  - Frame drops
  - App freezing
  - Out-of-memory crashes

### 3ï¸âƒ£ Built-in Optimizations

**FlatList** provides many props that automatically optimize rendering:

- `keyExtractor`
- `getItemLayout`
- `initialNumToRender`
- `maxToRenderPerBatch`
- `onEndReached` (for pagination)
- `windowSize`

**ScrollView** has none of these list-specific optimizations.

---

**FlatList** supports:

- Pull to refresh (`refreshing`, `onRefresh`)
- Pagination / infinite scroll (`onEndReached`)
- Separators, headers, and footers

**ScrollView** is more static and doesn't offer list-specific features.

## When Should You Use ScrollView?

Use **ScrollView** only if:

- You have small, static content
- The number of items is very limited (e.g., 5â€“10 elements)
- You need simple vertical scrolling with mixed content

## Interview Summary

**If the interviewer asks:**

> **Why is FlatList better than ScrollView?**

**Answer:**

> FlatList is better for large lists because it uses **virtualization** to render only visible items, making it more memory-efficient and performant. It also supports built-in features like pull-to-refresh, pagination, and item layout optimization, which ScrollView does not provide.
