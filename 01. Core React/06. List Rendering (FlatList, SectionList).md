# List Rendering (FlatList, SectionList) ‚Äî Interview-Ready Explanation

Rendering lists of data efficiently is a core concept in both React (web) and React Native (mobile).
But while React uses normal array methods like `.map()`, React Native provides optimized components like **FlatList** and **SectionList** designed for performance on mobile devices.

---

## ‚úÖ 1. List Rendering in React (Web)

In React (web), we usually render lists using `.map()`:

```jsx
const users = ['John', 'Sarah', 'Alex'];

export default function App() {
  return (
    <ul>
      {users.map((user, index) => (
        <li key={index}>{user}</li>
      ))}
    </ul>
  );
}
```

### üîë Key Points for Interview

- Must use unique `key` prop to help React identify items.
- Avoid using `index` as key unless no unique ID is available.
- Rerendering performance depends on proper keys and memoization.

---

## ‚úÖ 2. List Rendering in React Native

Unlike React web, React Native doesn't use `<ul>` or `.map()` directly for large lists.
**Why?** Performance issues. Mobile devices need **virtualization**.

---

## üß± FlatList ‚Äî Most Common List Component

**FlatList** is an optimized component for rendering large, uniform, scrollable lists.

### Example:

```jsx
const data = [
  { id: '1', title: 'Apple' },
  { id: '2', title: 'Banana' },
];

export default function App() {
  return (
    <FlatList
      data={data}
      keyExtractor={(item) => item.id}
      renderItem={({ item }) => <Text>{item.title}</Text>}
    />
  );
}
```

### üî• Why FlatList is important?

It uses **virtualization**:

- Only renders items visible on the screen.
- Recycles off-screen items to boost performance.
- Supports lazy loading, batch rendering, windowing.

### üîë Key Props

| Prop                     | Purpose                     |
| ------------------------ | --------------------------- |
| `data`                   | Array of items              |
| `renderItem`             | Function to render each row |
| `keyExtractor`           | Unique key for each item    |
| `horizontal`             | Horizontal scrolling        |
| `numColumns`             | Grid view support           |
| `ListHeaderComponent`    | Custom header               |
| `ListFooterComponent`    | Custom footer               |
| `ItemSeparatorComponent` | Divider                     |

### ‚ö° Performance Tips (Interview)

- Always provide **stable keys**.
- Use `React.memo` for row components.
- Use `getItemLayout` when item height is fixed ‚Üí improved scroll performance.
- Avoid inline functions inside `renderItem` ‚Üí use `useCallback`.

## üß± SectionList ‚Äî For Grouped/Sectioned Data

Used to render lists with sections, like contact lists grouped by alphabets.

### Example:

```jsx
const sections = [
  { title: 'Fruits', data: ['Apple', 'Banana'] },
  { title: 'Vegetables', data: ['Carrot', 'Spinach'] },
];

export default function App() {
  return (
    <SectionList
      sections={sections}
      keyExtractor={(item, index) => item + index}
      renderItem={({ item }) => <Text>{item}</Text>}
      renderSectionHeader={({ section }) => (
        <Text style={{ fontWeight: 'bold' }}>{section.title}</Text>
      )}
    />
  );
}
```

### üîë Key Props

| Prop                        | Purpose                |
| --------------------------- | ---------------------- |
| `sections`                  | Array of grouped items |
| `renderItem`                | Render each item       |
| `renderSectionHeader`       | Render section heading |
| `SectionSeparatorComponent` | Gap between sections   |

### Use When

Rendering **grouped data**  
e.g., categories, dates, alphabetical groups.

---

## üî• FlatList vs SectionList (Interview Summary)

| Feature       | FlatList          | SectionList                |
| ------------- | ----------------- | -------------------------- |
| Data type     | flat array        | array of sections          |
| Use case      | long simple lists | grouped lists              |
| Header/footer | yes               | yes (plus section headers) |
| Performance   | high              | high                       |

## üéØ Interview Tips & Questions to Prepare

### Q1. Why use FlatList instead of .map() in React Native?

Because **FlatList is virtualized**, meaning it only renders visible items ‚Üí more memory-efficient on mobile.

### Q2. What is virtualization?

Rendering only visible items; recycle items leaving the viewport.

### Q3. How to improve FlatList performance?

- Use `keyExtractor`
- Use `React.memo`
- Use `getItemLayout`
- Avoid inline arrow functions

### Q4. Difference between FlatList and SectionList?

**SectionList = FlatList + grouped sections support.**

---

## ‚úîÔ∏è Final Summary (Short Answer for Interviews)

- In **React**, list rendering uses `.map()`.
- In **React Native**, use **FlatList** or **SectionList** for efficient, virtualized, scrollable lists.

**FlatList** ‚Üí simple lists  
**SectionList** ‚Üí grouped lists

Both are highly optimized for performance.
