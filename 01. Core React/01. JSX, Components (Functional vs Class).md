## âœ… 1. JSX (JavaScript XML)

### What is JSX?

JSX is a syntax extension for JavaScript that looks like HTML but compiles to `React.createElement` calls.

**Example:**

```jsx
const element = <Text>Hello</Text>;
```

JSX makes the UI easier to write and understand because it lets you define UI structure directly inside JavaScript.

### Why JSX is important?

- It keeps UI and logic in one place (component-based architecture).
- It prevents errors with compile-time checks.
- It allows embedding expressions using `{}`.
- It supports conditional rendering:

```jsx
{
  isLoggedIn ? <Home /> : <Login />;
}
```

### JSX in React vs React Native

| Concept  | React (Web)                    | React Native                                      |
| -------- | ------------------------------ | ------------------------------------------------- |
| JSX tags | HTML-like (`div`, `p`, `span`) | Native components (`View`, `Text`, `Image`)       |
| Styling  | CSS                            | Stylesheet / inline styles (similar to CSS-in-JS) |
| Output   | DOM Elements                   | Native UI elements (bridged via native modules)   |

## âœ… 3. Functional Components

Functional components are JavaScript functions returning JSX.

**Example:**

```jsx
function Welcome({ name }) {
  return <Text>Hello {name}</Text>;
}
```

### Why functional components dominate modern React?

Because of **Hooks**.

Hooks provide:

- `useState` â†’ State
- `useEffect` â†’ Lifecycle
- `useContext` â†’ Context API
- `useRef` â†’ Referencing elements
- `useMemo` / `useCallback` â†’ Performance

### Functional component lifecycle mapping:

| Functional Component Hook              | Class Method Equivalent |
| -------------------------------------- | ----------------------- |
| `useEffect(() => {}, [])`              | `componentDidMount`     |
| `useEffect(() => { return () => {} })` | `componentWillUnmount`  |
| `useEffect(() => {}, [dep])`           | `componentDidUpdate`    |

### Advantages:

- Cleaner, simpler code
- Better performance due to no class instance
- Hooks make lifecycle/state easier
- Avoids `this` keyword confusion
- Recommended by React team

## âœ… 4. Class Components

Older style components using ES6 classes. Rarely used now but important for interviews.

**Example:**

```jsx
class Welcome extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  componentDidMount() {
    console.log('Mounted');
  }

  render() {
    return (
      <Text onPress={() => this.setState({ count: this.state.count + 1 })}>
        Count: {this.state.count}
      </Text>
    );
  }
}
```

### Features:

- Uses `this.state`
- Lifecycle methods:
  - `componentDidMount`
  - `componentDidUpdate`
  - `componentWillUnmount`
- Binding: Must bind functions sometimes:
  ```jsx
  this.handlePress = this.handlePress.bind(this);
  ```

### Disadvantages:

- Verbose
- More boilerplate
- `this` keyword confusion
- Hooks made them almost obsolete
