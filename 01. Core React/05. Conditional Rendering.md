## âš¡ Common Techniques for Conditional Rendering

### 1. if / else (classic way)

Useful when the logic is long or complex.

```jsx
if (!user) {
  return <Text>Please login</Text>;
}

return <Text>Welcome {user.name}</Text>;
```

**When to use?**

- Long logic
- Multiple return cases
- Readability matters

### 2. Ternary Operator (most popular)

Great for inline conditions inside JSX.

```jsx
{
  isLoading ? <ActivityIndicator /> : <Text>Data Loaded</Text>;
}
```

**When to use?**

- Simple A or B decisions
- Loading vs data

### 3. Logical AND `&&`

Used when you want to show something **only when** the condition is true.

```jsx
{
  showMessage && <Text>Hello User!</Text>;
}
```

**When to use?**

- Show a small component conditionally
- Show/hide UI blocks

âš ï¸ **Note:** Avoid using `0 && ...` because `0` is falsy and may cause unexpected output.

### 4. Using Functions for Conditional Rendering

Good for separating logic from UI.

```jsx
const renderContent = () => {
  if (error) return <Text>Error loading data</Text>;
  if (loading) return <ActivityIndicator />;
  return <Text>Data Ready</Text>;
};

return <View>{renderContent()}</View>;
```

**When to use?**

- To keep JSX clean
- Multiple complex states

### 5. Switch Case Rendering

Good when you have multiple modes/states.

```jsx
switch (status) {
  case 'loading':
    return <ActivityIndicator />;
  case 'error':
    return <Text>Error...</Text>;
  default:
    return <Text>Success!</Text>;
}
```

## ðŸ”¥ Real-world Interview Examples

### Q1: How do you show a loader while data is fetching?

```jsx
return <View>{loading ? <ActivityIndicator /> : <Text>{data}</Text>}</View>;
```

### Q2: How do you conditionally apply styles?

```jsx
<Text style={isError ? styles.error : styles.normal}>Message</Text>
```

### Q3: How do you conditionally render multiple UI states?

```jsx
const renderUI = () => {
  if (loading) return <ActivityIndicator />;
  if (error) return <Text>Error!</Text>;
  if (!data) return <Text>No Data Found</Text>;
  return <Text>Data: {data}</Text>;
};

return <View>{renderUI()}</View>;
```
